# GSD Command Reference for Antigravity

**GSD** (Get Shit Done) is a spec-driven development system optimized for agentic coding.

## Core Setup
- `/gsd-map-codebase` - Analyze existing code and document its architecture/stack.
- `/gsd-new-project` - Initialize a new project, conduct research, and define requirements.

## Planning & Execution
- `/gsd-plan-phase <N>` - Create a detailed execution plan for Phase N.
- `/gsd-execute-phase <N>` - Orchestrate the execution of all plans in Phase N.
- `/gsd-progress` - Check project status and see what's next.

## Tasks & Debugging
- `/gsd-add-todo` - Capture an idea or task as a structured todo.
- `/gsd-check-todos` - Review pending todos and select one to work on.
- `/gsd-debug` - Systematic investigation and fixing of issues.

## Management
- `/gsd-settings` - Configure workflow preferences.
- `/gsd-complete-milestone` - Archive current work and prepare for the next version.

---
*For best results, run `/clear` between major GSD phases to keep context windows clean.*
